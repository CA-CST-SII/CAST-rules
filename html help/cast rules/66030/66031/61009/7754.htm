Avoid using COMPUTE statement for elementary arithmetic operation

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

COMPUTE is a performance consuming statement and should be used to perform complex arithmetic operations. Simple operation should be performed by using ADD, SUBTRACT, MULTIPLY and DIVIDE statements that are more efficient and more easy to understand.

Description

This rule searches for Cobol programs using the COMPUTE statement for elementary arithmetic operation. Expressions with only one binary operator (+, -, *, /) are taken into account. Expressions based on the exponentiation operator ('**') are not considered as violating the rule. Expression with only one operand (with or without an unary operator) are taken into account. Example: - A = B - A = -1 - A = 0

Remediation

Replace COMPUTE statements used for only one arithmetic operation by ADD, SUBTRACT, MULTIPLY or DIVIDE statements.

Reference

IBM Enterprise Cobol - Programming Guide

Sample
COMPUTE A = A + 1
COMPUTE D = B / C
COMPUTE E = 4
Remediation sample
ADD 1 TO A
DIVIDE B BY C GIVING D
MOVE 4 TO E
