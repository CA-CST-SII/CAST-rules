Avoid empty catch blocks (C# .Net)

Technology

.Net, C#

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

6

Rationale

An empty catch block defeats the purpose of exceptions. When an exception occurs, nothing happens and the program fails for an unknown reason. The application can be in an unknown state that will affect subsequent processing. Since the reason for the issue (the type of the exception and potential embedded message) are ignored, it will require more time to fix the issue.

Description

This Quality Rule reports all methods with at least one empty catch block (empty or containing only comments). In a try and catch statement, Catch blocks should contain code to handle the thrown exception. If they are empty or contain only comments, the Exception will not be handled.

Remediation

The exception must be handled correctly according to its type.

Sample
try { ... }
catch ( MyException e)
{
   // DO NOTHING <= VIOLATION
}
Remediation sample
try { ,,, }
catch ( MyException e)
{
   DoSomething();
}
