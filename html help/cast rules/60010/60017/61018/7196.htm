Avoid large number of String concatenation

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

String concatenation resolved at runtime is much slower than using StringBuffer.

Description

All String Classes should not call more than X times the + Method. This does not take into account literals since they are automatically optimized by the compiler. For example: String a = "abc"+"abcd" will not be counted as a concatenation.

Remediation

Use StringBuffer and StringBuffer.append() method instead

Reference

http://www.precisejava.com/javaperf/j2se/StringAndStringBuffer.htm#Strings104

Sample
String result = "hello";
result += " John";
result += " Smith"; // VIOLATION if the threshold is 3
Remediation sample
StringBuffer result = new StringBuffer("hello");
result.append(" John");
result.append(" Smith");
