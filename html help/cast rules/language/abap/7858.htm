Avoid passing parameter by value

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

The possible forms of parameter transfer in ABAP are pass by reference (indicated by Reference(PARAM) or not at all) and pass by value (indicated by Value(PARAM) ). The general rule is that pass by value is always slower than pass by reference. The loss in performance is always a minimum of 40 % and can rise to several 1000 %, depending on the data type and data volume.

Description

This rule checks all Abap subroutines (Form) and find out those that define formal parameters with the VALUE() addition.

Remediation

Try to avoid passing parameter by value as much as possible.

Sample
DATA: BEGIN OF X.
INCLUDE STRUCTURE SFLIGHT.
DATA: ADDITION(8) TYPE C,
END OF X.
...
PERFORM U USING X.
...
FORM U USING VALUE(X) STRUCTURE SFLIGHT.
WRITE: X-FLDATE.
ENDFORM.
Remediation sample

INCLUDE STRUCTURE SFLIGHT.
DATA: ADDITION(8) TYPE C,
