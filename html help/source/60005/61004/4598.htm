Rationale

For portability reasons, 'java.lang.Runtime.exec()' should not be used since it means being dependant on the environment where the application is deployed. For security reasons, 'java.lang.Runtime.exec()' can lead to malicious file execution resulting in devastating attacks such as total server compromise. There are uses correct uses of Runtime.exec for example when the method call is platform neutral. Such examples of the correct use of Runtime.exec are: - Invocation of a Java compiler, with the name of the compiler specified as a user-settable Property. - Execution of a command the user typed in (a "shell"). - Invocation of a browser, configured as part of the installation of the program, when the user presses a "Help" button.

Description

Java artifacts should not use 'java.lang.Runtime.exec()'
