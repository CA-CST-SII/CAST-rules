Avoid multiple validation form with the same name

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

9

Rationale

When several validation form have the same name, the Struts Validator framework will choose one of the form validation to check input values. This means that the behavior is not controlled and that wrong input pass the validation check. This may lead to SQL injection, cross-site scripting...

Description

This metric reports all duplicated form validators with the same name. Note that it can be the form name or the action name (in the case of a form shared among different actions)

Remediation

Find the right validator and remove the others.

Reference

http://cwe.mitre.org/data/definitions/102.html http://www.owasp.org/index.php/Struts:_Duplicate_Validation_Forms

Sample
----> validation.xml:
<form-validation>
  <formset>
    <form name="logonBean">
    ...
    </form>
    <form name="logonBean"> // VIOLATION
    ...
    </form>
  </formset>
</form-validation>
Remediation sample
----> validation.xml:
<form-validation>
  <formset>
    <form name="logonBean"> // FIXED
    ...
    </form>
  </formset>
</form-validation>
Output

This report lists all validation forms that are duplicated. It provides the following information: Form Validation Full Name

Total

The total number of distinct validation forms names or 0 if no object of type "Validation Configuration File" is present
