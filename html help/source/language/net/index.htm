Specific information for all .NET Quality Rules

All .NET Quality Rules can list objects as Very High Risk Objects that appear to be generated by CAST but that are not.

For example:

Diagnostic: C#: Avoid unreferenced Data Members

Object (C# Field): uc_infospersobasecontact_ascx_CastGenerated_bf1e440e.divSaisieAdresse3

Explanation: For ASP.Net applications, some source code elements of ASP.net pages are attached to a class object that is not present in the source code and named "CastGenerated". The reason for this is to mimic the behaviour of .NET. These objects are not generated. They are user code attached to an object that is named "CastGenerated".
