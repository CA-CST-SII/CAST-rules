Rationale

By using Stored Procedures the database engine is more able to optimize the access plan and to reuse them. It also limit the parsing phase of the SQL order. This generally result in better performance. From a security point of view, it is generally safer to use SP rather than dynamic SQL as this limit the risk of having SQL-injection.

Description

Data Access must be based on Stored Procedure Calls.

Remediation

Transform the SQL into a SP and use parameters. Then call the SP. Do not transform the SQL in a SP that in turn uses dynamic SQL (e.g. @exec or EXECUTE_IMMEDIATE) as this deny all the benefits of the change.

Sample
class MyClass
{
   private String s = "select col1from MyTable where col2 =";
   public void fct( String sVal )
   {
      s +=  "'" + sVal + "'"; 
      exec s;
   }
Remediation sample
class MyClass
{
   public void fct( String sVal )
   {
      exec "MySP +sVal"
   }
