Avoid triggers, functions and procedures with a very low comment/code ratio

Rationale

Maintenability of the code is facilitated if there is documentation in the code. This rule will ensure there are enough comments within triggers, functions and procedures. For PL/SQL code the functions and procedures that are declared in packages is also concerned.

Description

Triggers, functions and procedures must have a comment/code ratio (%) greater than the set threshold value (%) Note, threshold is a parameter and can be changed at will.

Remediation

Enrich triggers, functions and procedures code with comments

Output

Associated to each Trigger/Function/Procedure with violations, the Quality Rule provides: - The comment / code ratio
