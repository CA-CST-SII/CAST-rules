Avoid Constructors not supplying an initial value for all non-static data members

Technology

C++

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

1

Rationale

Each constructor must initialize members defined with a scalar type or that are pointers in order to reduce the risk of an invalid state after successful construction and to ease the understanding of the code. Not initializing scalar members and pointers can lead to unexpected behavior.

Description

This rule searches for C++ class constructors that do not initialize all class members defined with a scalar type or that are pointers. Initializations can be created in the constructor declaration or in the constructor body. Class members that are not taken into account are: - Class instances - Defined with a non-scalar type

Remediation

Initialize scalar-typed members in constructor header.

Reference

Effective C++, Item 12 Industrial Strength C++ 5.5

Sample
class C
{
  public:
     C() {  }; // VIOLATION: m_i is not initialized
  protected:
     int m_i;
}
Remediation sample
// Remediation 
class C
{
  public:
     C() : m_i(0) { }
  protected:
     int m_i;
}
