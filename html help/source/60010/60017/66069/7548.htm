Never use incompatible statements with the CICS environment

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

9

Rationale

The CICS environment forbids some Cobol statements concerning I/O operations and program termination. The use of these statements can cause problems in production and can crash applications.

Description

This rule searches for Cobol programs running under the CICS environment containing statements that are not permitted. These statements are the following: - ACCEPT - ENTRY - DISPLAY - FD and SD - SELECT <file> ASSIGN TO - OPEN, CLOSE, READ, WRITE, REWRITE, DELETE - MERGE, SORT - START - STOP The following exception is taken into account: some batch programs can contain EXEC CICS commands in order to run CICS utilities (DFH*). If these types of programs do not execute other commands, then they will not be considered as violating the rule.

Remediation

These statements must be removed from programs running under the CICS environment. Files can be accessed by using specific macros provided with CICS.

Reference

IBM Enterprise Cobol for z/OS - Programming Guide
