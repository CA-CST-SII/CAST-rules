Avoid SQL queries that no index can support

Rationale

SQL Queries that do not use table indexes are extremely dangerous for the application's performance. Indeed, each execution of the query will result in a full table scan which is extremely time consuming.

Description

This metric retrieves the artifacts containing at least one SQL Query not using a table's indexes. An SQL Query is using table's indexes if all the following conditions are true: - At least one index is defined for each table participating in the from list - Queries must reference left-most columns of the index key

Remediation

Try to use indexed columns in WHERE and HAVING clauses. If it is not possible, then create a secondary index for these columns.
