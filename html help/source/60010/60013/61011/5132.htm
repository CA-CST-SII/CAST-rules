Avoid Programs with a Complex SELECT Clause (Cobol)

Technology

Cobol

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

Complex SELECT clauses, i.e select clauses with many column can be difficult to read and does not help the reader who needs to identify the relevant columns to be retrieved. Also a query that retrieves many columns can potentially cause performance problems: Such performance problems may arise when the execution of the query returns a large result sets (many row with many columns may then become a huge amount of data to transport over the network).

Description

Avoid Cobol Programs with a SELECT clause returning more than 9 columns. Such queries are considered complex. Changing threshold value requires a Metric Assistant configuration update.

Remediation

Review the SELECT statement to reduce the number of selected columns.

Output

Associated to each COBOL Program with violations, the Quality Rule provides: - The number of complex SELECT clauses
