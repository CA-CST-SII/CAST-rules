Avoid hiding attributes (JEE)

Technology

JEE

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

Hiding attributes is not good Object Oriented practice. It is a redefinition of an Instance or Class variable and it makes the relationship between the parent and the child class unclear. It is also likely to be an unintended mistake.

Description

Hiding attributes is not allowed. An attribute <attr> of Class <SuperClass> is overridden in SubClass <SubClass> if <SubClass> contains an identical declaration of <attr> (i.e. same name, the type can be different) except when the attribute is serialVersionID or when its type is of type or a subclass of 'org.apache.log4j.Logger' or org.apache.commons.logging.Log.

Remediation

The solution depends on the design: - use the inherited member and remove the member declaration in the child class - rename the member of the child class

Sample
public class SampleBase {
     protected long a = 4;
 }
 
 public class Sample extends SampleBase {
    protected int a = 5;  // VIOLATION
 }
